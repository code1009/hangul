# 한글 코드의 역사

한글은 초성, 중성, 종성이 결합하여 하나의 글자를 만드는 **모아쓰기** 방식의 문자입니다.

이 독특한 구조를 컴퓨터의 이진수 체계에 담기 위해

지난 수십 년간 수많은 표준과 코드 체계가 경쟁하고 변화해 왔습니다.

---

## 한글 코드 제정 연대표

- 1987년: KS C 5601-1987 (완성형 한글) 제정
- 1992년: KS C 5601-1992 (조합형 한글 부속서 추가)
- 1995년: 마이크로소프트 CP949(확장 완성형/UHC) 도입 (윈도우95)
- 1996년: 유니코드 2.0에 현대 한글 11,172자 공식 수록 (U+AC00~U+D7A3)
- 2000년: KS X 1001로 표준명 변경 (기존 KS C 5601)
- 2004년: KS X 1005-1(유니코드) 국가표준 채택

---

## 조합형 한글

한글의 창제 원리를 가장 충실히 반영한 방식입니다.

초성, 중성, 종성을 각각의 비트로 할당하여 실시간으로 조합해 화면에 뿌려줍니다.

* **구조 (2바이트 / 16비트):**
  * **1비트:** 한글/영문 구분 (항상 1)
  * **5비트 (초성):** ㄱ, ㄴ, ㄷ... (32가지 상태)
  * **5비트 (중성):** ㅏ, ㅑ, ㅓ... (32가지 상태)
  * **5비트 (종성):** (받침 없음 포함) ㄱ, ㄴ, ㄷ... (32가지 상태)
* **국가 표준:** **KS C 5601-1992 (부속서 3)**에 '상용 조합형'으로 채택 (현 **KS X 1001** 부속서).
* **코드페이지 (CP):** **CP1361** (마이크로소프트 할당).
* **특징:**
  * **장점:** 현대 한글로 표현 가능한 **11,172자** 전체를 제약 없이 표현 가능.
  * **주요 사례:** MS-DOS 시절의 '아래아 한글', '보석글' 등.

---

## 완성형 한글

한글 한 글자를 하나의 완성된 덩어리(코드)로 간주하는 방식입니다.

한자나 특수문자를 처리하는 방식과 유사합니다.

* **국가 표준:** **KS X 1001** (구 **KS C 5601-1987**).
* **코드페이지 (CP):** **CP949** (초기 버전).
* **구조:** 2바이트를 사용하며 구(Row)와 점(Column)의 행렬 구조로 배치.
* **특징:**
  * **장점:** 시스템 안정성이 높고 영문과의 혼용 시 글자 깨짐이 적음.
  * **단점:** 표준에 등록된 **2,350자**의 한글만 지원. '똠', '훵', '뷁' 같은 글자는 입력 불가능.

---

## 확장 완성형 (UHC: Unified Hangul Code)

완성형의 치명적인 한계(글자 수 부족)를 해결하기 위해 마이크로소프트가 독자적으로 고안한 방식입니다.

* **코드페이지 (CP):** **CP949** (현재 윈도우 한글 환경의 기본값).
* **구조:** 기존 완성형(KS X 1001)의 빈 공간에 나머지 8,822자를 강제로 배치.
* **특징:**
  * 국가 표준은 아니나 윈도우 95 보급으로 사실상의 표준이 됨.
  * 기존 완성형과 하위 호환성을 유지하면서 11,172자를 모두 수용.

## 유니코드

전 세계 모든 문자를 하나의 코드 체계 안에 통합하려는 국제 표준입니다.

* **국가 표준:** **KS X 1005-1** (국제 표준 ISO/IEC 10646 수용).
* **코드페이지 (CP):** * **CP65001:** UTF-8 (가변 길이 인코딩, 웹 표준).
  * **CP1200:** UTF-16 (윈도우 내부 처리 기본값).
* **한글 수용:** 유니코드 2.0(1996년)에서 현대 한글 **11,172자** 가나다순 배치 (`U+AC00` ~ `U+D7A3`).
* **특징:** 옛 한글(고어)까지 자모 조합 방식으로 완벽 지원.
