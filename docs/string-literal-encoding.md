# 컴파일러별 문자열 리터럴 인코딩 처리

소스 파일의 인코딩(저장 방식)과 컴파일 결과물에 담기는 바이트(실행 문자셋)는 별개 개념이다.  
같은 UTF-8 소스 파일이라도 컴파일러 설정에 따라 `""`, `L""` 리터럴의 바이트 표현이 달라진다.

---

## 용어 정리

| 용어 | 설명 |
|------|------|
| **소스 문자셋** (source charset) | 컴파일러가 소스 파일을 읽을 때 사용하는 인코딩 |
| **실행 문자셋** (execution charset) | 컴파일 결과물(바이너리)에서 `""` 리터럴이 담기는 인코딩 |
| **와이드 문자셋** (wide charset) | `L""` 리터럴이 담기는 인코딩 (`wchar_t` 크기에 따라 결정) |

---

## MSVC (Microsoft Visual C++)

### `wchar_t` 크기

Windows에서 `wchar_t`는 **2바이트** → `L""` 는 항상 **UTF-16 LE**

### 기본 동작 (플래그 없음)

| 소스 파일 인코딩 | 리터럴 | 컴파일 결과 인코딩 |
|----------------|--------|-------------------|
| 임의 (BOM 없는 UTF-8 등) | `L"한글"` | UTF-16 LE |
| 임의 | `"한글"` | 시스템 ANSI 코드페이지 (한국어 Windows = **CP949**) |

- 기본 소스 문자셋: 시스템 코드페이지 (CP949)
- 기본 실행 문자셋: 시스템 코드페이지 (CP949)
- **UTF-8 BOM이 있으면** MSVC가 소스를 UTF-8로 읽지만, 실행 문자셋은 여전히 CP949 → `"한글"`은 CP949로 변환되어 저장됨

### UTF-8 BOM 없는 UTF-8 소스의 위험

```
소스 파일: UTF-8 저장, BOM 없음
  →  MSVC가 CP949로 읽으려 시도
  →  멀티바이트 경계가 맞지 않아 문자 깨짐 또는 경고 발생
```

### 플래그로 동작 변경

| 플래그 | 효과 |
|--------|------|
| `/utf-8` | 소스 문자셋 + 실행 문자셋 모두 UTF-8 |
| `/source-charset:utf-8` | 소스 문자셋만 UTF-8로 |
| `/execution-charset:utf-8` | 실행 문자셋만 UTF-8로 |

```
/utf-8 플래그 적용 시:
  "한글"  →  UTF-8 바이트 (0xED 0x95 0x9C 0xAE 0x80)
  L"한글" →  UTF-16 LE (변화 없음)
```

### C++11 이후 접두사별 동작

| 리터럴 | 타입 | 인코딩 |
|--------|------|--------|
| `"..."` | `char[]` | 실행 문자셋 (기본 CP949) |
| `L"..."` | `wchar_t[]` | UTF-16 LE |
| `u8"..."` | `char[]` / `char8_t[]` | **항상 UTF-8** (C++11~) |
| `u"..."` | `char16_t[]` | **항상 UTF-16** |
| `U"..."` | `char32_t[]` | **항상 UTF-32** |

## GCC / G++

### `wchar_t` 크기 (플랫폼에 따라 다름)

| 플랫폼 | `wchar_t` 크기 | `L""` 인코딩 |
|--------|:--------------:|:------------:|
| Linux (glibc) | **4바이트** | **UTF-32 LE** |
| Windows MinGW | **2바이트** | **UTF-16 LE** |

### 기본 동작

GCC는 소스 문자셋과 실행 문자셋을 각각 `-finput-charset`, `-fexec-charset` 플래그로 제어한다.  
**기본값은 둘 다 UTF-8** (현대 Linux 기준).

| 소스 파일 인코딩 | 리터럴 | Linux 결과 | MinGW 결과 |
|----------------|--------|------------|------------|
| UTF-8 | `"한글"` | UTF-8 | UTF-8 |
| UTF-8 | `L"한글"` | UTF-32 LE | UTF-16 LE |
| EUC-KR | `"한글"` | UTF-8 변환 후 저장¹ | UTF-8 변환 후 저장¹ |

> ¹ GCC는 소스를 `-finput-charset`으로 읽은 뒤 `-fexec-charset`으로 변환하여 바이너리에 저장한다.

### 플래그 예시

```bash
# 소스가 EUC-KR로 저장된 경우
gcc -finput-charset=EUC-KR -fexec-charset=UTF-8 source.c

# 기본값 명시 (UTF-8 소스 → UTF-8 결과물)
gcc -finput-charset=UTF-8 -fexec-charset=UTF-8 source.c
```

### C++11 이후 접두사별 동작

MSVC와 동일하게 `u8""`, `u""`, `U""` 접두사는 인코딩이 고정된다.

| 리터럴 | 타입 | 인코딩 |
|--------|------|--------|
| `"..."` | `char[]` | 실행 문자셋 (기본 UTF-8) |
| `L"..."` | `wchar_t[]` | UTF-32 (Linux) / UTF-16 (MinGW) |
| `u8"..."` | `char[]` / `char8_t[]` | **항상 UTF-8** |
| `u"..."` | `char16_t[]` | **항상 UTF-16** |
| `U"..."` | `char32_t[]` | **항상 UTF-32** |

## 컴파일러별 비교 요약

| 조건 | MSVC (Korean Windows) | GCC (Linux) | GCC (MinGW/Windows) |
|------|-----------------------|-------------|----------------------|
| 기본 소스 문자셋 | CP949 | UTF-8 | UTF-8 |
| 기본 실행 문자셋 | CP949 | UTF-8 | UTF-8 |
| `"한글"` 기본 | CP949 `(0xC7 0xD1 0xB1 0xDB)` | UTF-8 `(0xED 0x95 0x9C 0xAE 0x80)` | UTF-8 |
| `L"한글"` | UTF-16 LE `(0xAC00...)` | UTF-32 LE `(0x00AC00...)` | UTF-16 LE `(0xAC00...)` |
| `wchar_t` 크기 | 2바이트 | 4바이트 | 2바이트 |
| UTF-8 강제 방법 | `/utf-8` 플래그 | 기본값이 이미 UTF-8 | 기본값이 이미 UTF-8 |

## 실무 권장 사항

1. **이식성이 필요한 코드**: `u8""`, `u""`, `U""` 접두사를 사용하여 인코딩을 명시한다.
2. **MSVC + 한글 소스**: UTF-8 BOM으로 저장하거나 `/utf-8` 플래그를 프로젝트 설정에 추가한다.
3. **Windows API 호출**: `wchar_t` / `L""` (UTF-16 LE)를 사용하거나 `MultiByteToWideChar` / `WideCharToMultiByte`로 변환한다.
4. **Linux/GCC 크로스 컴파일**: `L""` 대신 `u""` (`char16_t`)를 사용하면 플랫폼 무관하게 UTF-16이 보장된다.
