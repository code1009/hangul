# 한글 유니코드

## 한글 코드

### 한글 11172자

```
(초성 19자) * (중성 21자) * (종성 28자) = 11172자
```

## UTF-8 인코딩

UTF-8은 코드포인트 범위에 따라 1~4바이트로 인코딩한다.

| 코드포인트 범위 | 바이트 수 | 바이트 패턴 |
|----------------|:---------:|-------------|
| U+0000 ~ U+007F | 1 | `0xxxxxxx` |
| U+0080 ~ U+07FF | 2 | `110xxxxx 10xxxxxx` |
| U+0800 ~ U+FFFF | 3 | `1110xxxx 10xxxxxx 10xxxxxx` |
| U+10000 ~ U+10FFFF | 4 | `11110xxx 10xxxxxx 10xxxxxx 10xxxxxx` |

한글 음절(U+AC00–U+D7A3), 자모(U+1100–U+11FF, U+3131–U+3163)는 모두 U+0800–U+FFFF 범위에 속하므로 **3바이트**로 인코딩된다.

### UTF-8 인코딩 예시 (3바이트)

코드포인트 `U+HHHH`를 이진수로 나타내면 `1111_2222_3333_4444`:

```
1110_1111  10_222233  10_333344
  0xEF       0xB0...    0x80...
```

| 문자 | 코드포인트 | 이진수 | UTF-8 바이트 |
|:----:|:----------:|--------|:------------:|
| 가 | U+AC00 | `1010 110000 000000` | `0xEA 0xB0 0x80` |
| 나 | U+B098 | `1011 000010 011000` | `0xEB 0x82 0x98` |
| 힣 | U+D7A3 | `1101 011110 100011` | `0xED 0x9E 0xA3` |
| ㄱ | U+3131 | `0011 000101 110001` | `0xE3 0x84 0xB1` |

### UTF-8 인코딩 상세 (가, U+AC00)

```
U+AC00 = 0xAC00 = 0b 1010 1100 0000 0000

3바이트 패턴: 1110xxxx 10xxxxxx 10xxxxxx
                   ^^^^   ^^^^^^   ^^^^^^
비트 분리:         1010   110000   000000

결과:       11101010 10110000 10000000
              0xEA     0xB0     0x80
```

## UTF-16 인코딩

UTF-16은 코드포인트에 따라 2바이트(BMP) 또는 4바이트(서러게이트 쌍)로 인코딩한다.

| 코드포인트 범위 | 바이트 수 | 방식 |
|----------------|:---------:|------|
| U+0000 ~ U+D7FF, U+E000 ~ U+FFFF | 2 | 코드포인트 값 그대로 |
| U+10000 ~ U+10FFFF | 4 | 서러게이트 쌍 (surrogate pair) |

한글 음절(U+AC00–U+D7A3), 자모(U+1100–U+11FF, U+3131–U+3163)는 모두 BMP(기본 다국어 평면, U+0000–U+FFFF) 내에 있으므로 서러게이트 쌍 없이 **2바이트**로 인코딩된다.

### 바이트 순서 (BOM)

| 방식 | 바이트 저장 순서 | BOM |
|------|----------------|-----|
| UTF-16 LE (Little-Endian) | 하위 바이트 먼저 | `0xFF 0xFE` |
| UTF-16 BE (Big-Endian) | 상위 바이트 먼저 | `0xFE 0xFF` |

Windows `wchar_t`는 UTF-16 LE를 사용한다.

### UTF-16 인코딩 예시

| 문자 | 코드포인트 | UTF-16 BE | UTF-16 LE |
|:----:|:----------:|:---------:|:---------:|
| 가 | U+AC00 | `0xAC 0x00` | `0x00 0xAC` |
| 나 | U+B098 | `0xB0 0x98` | `0x98 0xB0` |
| 힣 | U+D7A3 | `0xD7 0xA3` | `0xA3 0xD7` |
| ㄱ | U+3131 | `0x31 0x31` | `0x31 0x31` |

### 서러게이트 쌍 (U+10000 이상, 한글 해당 없음)

한글에는 해당하지 않으나 참고용으로:

```
코드포인트 U+HHHHH (U+10000 이상):
  U' = U - 0x10000          // 0x00000 ~ 0xFFFFF (20비트)
  상위 서러게이트 = 0xD800 + (U' >> 10)       // 0xD800 ~ 0xDBFF
  하위 서러게이트 = 0xDC00 + (U' & 0x3FF)     // 0xDC00 ~ 0xDFFF
```

## 유니코드 평면 (Plane)

유니코드 코드포인트 공간은 **17개 평면**으로 나뉘며, 각 평면은 65,536(0x10000)개의 코드포인트를 포함한다.

```
총 코드포인트 수 = 17 × 65,536 = 1,114,112개  (U+0000 ~ U+10FFFF)
                                               = 2^21 비트 주소 공간  (최대값 U+10FFFF = 0x10FFFF = 0001_0000_1111_1111_1111_1111₂)
```

> 단, **코드포인트 수 ≠ 할당된 문자 수**이다.  
> 서러게이트(2,048개), 비문자(66개), 사용자 정의 영역(137,468개), 미할당 영역 등을 제외한  
> **실제 할당된 문자 수는 유니코드 버전마다 다르며**, Unicode 15.1 (2023) 기준 약 **149,813자**다.

## 서러게이트(Surrogate)란?

### 개요

서러게이트는 UTF-16이 **BMP(U+0000–U+FFFF) 이상의 코드포인트**를 표현하기 위해 예약해 둔 코드포인트 범위다.  
유니코드 표준은 이 범위를 **문자로 직접 사용할 수 없도록** 영구 예약하고 있다.

| 범위 | 이름 | 용도 |
|------|------|------|
| U+D800 – U+DBFF | 상위 서러게이트 (High Surrogate) | 서러게이트 쌍의 첫 번째 코드 유닛 |
| U+DC00 – U+DFFF | 하위 서러게이트 (Low Surrogate) | 서러게이트 쌍의 두 번째 코드 유닛 |
| U+D800 – U+DFFF | 서러게이트 전체 | 총 2048개 코드포인트 예약 |

### 서러게이트가 필요한 이유

UTF-16은 코드 유닛 크기가 16비트(2바이트)이므로, 최대 65536(2¹⁶)가지 값만 직접 표현할 수 있다.  
그러나 유니코드 코드포인트 공간은 **U+0000–U+10FFFF** 로 총 **17개 평면** × 65,536 = 1,114,112개에 달한다.

```
BMP(U+0000–U+FFFF): 65,536자 → 2바이트로 직접 표현
보조 평면(U+10000–U+10FFFF): 1,048,576자 → 서러게이트 쌍 필요
```

이를 해결하기 위해 BMP 내의 2048개 코드포인트(U+D800–U+DFFF)를 "서러게이트"로 예약하고,  
상위·하위 서러게이트 한 쌍(4바이트)으로 보조 평면의 문자를 표현한다.

### 서러게이트 쌍 인코딩 공식

```
입력: 코드포인트 U  (U+10000 이상)

1. U' = U - 0x10000                 // 0x00000 ~ 0xFFFFF (20비트)
2. 상위 서러게이트 W1 = 0xD800 + (U' >> 10)      // bits 19~10
3. 하위 서러게이트 W2 = 0xDC00 + (U' & 0x3FF)    // bits  9~ 0
```

### 서러게이트 쌍 디코딩 공식

```
입력: W1 (0xD800–0xDBFF), W2 (0xDC00–0xDFFF)

U' = ((W1 - 0xD800) << 10) | (W2 - 0xDC00)
U  = U' + 0x10000
```

### 서러게이트 쌍 예시 (😀 U+1F600)

```
U = 0x1F600
U' = 0x1F600 - 0x10000 = 0xF600 = 0b 0000111101 1000000000

W1 = 0xD800 + 0b0000111101 = 0xD800 + 0x3D  = 0xD83D  (상위)
W2 = 0xDC00 + 0b1000000000 = 0xDC00 + 0x200 = 0xDE00  (하위)

UTF-16 LE 바이트: 0x3D 0xD8  0x00 0xDE
```

### 서러게이트와 UTF-8

UTF-8은 서러게이트 범위(U+D800–U+DFFF)를 **인코딩하지 않는다.**  
유니코드 표준에서 서러게이트 코드포인트는 문자가 아니므로, 올바른 UTF-8 시퀀스에 서러게이트 인코딩이 나타나면 **잘못된 UTF-8**로 간주한다.

> **CESU-8 / WTF-8**: 일부 시스템(Java 내부 표현 등)에서 서러게이트 쌍을 각각 3바이트 UTF-8로 인코딩하는 비표준 방식을 사용하기도 한다. 이는 정식 UTF-8이 아니다.

### 한글과 서러게이트

| 코드포인트 범위 | 서러게이트 필요 여부 |
